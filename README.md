# FlowDoc Editor

FlowDoc Editor 是一个基于 Electron 的本地 GUI 文档编辑器，交互方式参考飞书云文档和 Typora，强调“所见即所得”的实时编辑体验。

文档正文直接以排版结果呈现，标题、列表、引用、代码块、图片和附件都会在正文中即时渲染，而不是拆成“源码模式 + 预览模式”两套界面。

## 功能特性

### 文档管理

- 支持新建 `.flowdoc` 文档。
- 新建文档时必须先选择保存位置，随后才能进入编辑。
- 支持打开已有 `.flowdoc` 文档。
- 程序启动后会自动扫描桌面 `本地文档` 目录中的 `.flowdoc` 文件，并在侧边栏提供文档库检索。
- 文档主标题与文件名保持同步：
  - 修改文档首个 `h1` 会触发文件重命名。
  - 外部重命名文档后，编辑器重新获得焦点时会尝试刷新路径并继续保存到新文件名。

### 编辑体验

- 正文采用所见即所得实时编辑。
- 选中文本后会自动弹出悬浮格式菜单。
- 粘贴普通链接时会自动保留为可点击链接。
- 块级 `+` 按钮会跟随文本光标，并显示在光标上方。
- 支持：
  - 标题 1 / 标题 2 / 标题 3
  - 加粗 / 斜体
  - 有序列表 / 无序列表
  - 引用块
  - 代码块
- 块级 `+` 菜单会跟随当前编辑位置，用于插入图片、附件、代码块和引用块。
- 支持撤销 / 重做，并将历史快照持久化到本地存储中。

### 代码块

- 代码块支持实时语法高亮。
- 支持切换语言：
  - Plain Text
  - JavaScript
  - TypeScript
  - JSON
  - HTML / XML
  - CSS
  - Markdown
  - Bash
  - Python
  - SQL
- 代码块顶部带有 `Copy` 按钮，可一键复制整个代码块内容到系统剪贴板。
- 在代码块内：
  - `Tab` 会插入空格缩进
  - 第一次 `Ctrl+A` 选中当前代码块
  - 第二次 `Ctrl+A` 选中整篇文档

### 图片与附件

- 支持选择本地图片插入文档。
- 支持直接粘贴图片到正文中。
- 支持上传附件到文档中。
- 支持粘贴主流平台的视频嵌入代码，目前已适配 YouTube 和 Bilibili。
- 打开旧文档时会自动解析图片与附件节点。
- 如果对应资源文件缺失，会在正文中显示缺失占位提示，而不是直接失效。
- 附件节点支持点击后复制到当前用户的 `Downloads` 目录。

### 资源管理

- 所有图片统一存放在程序目录下的 `images/`。
- 所有附件统一存放在程序目录下的 `attachments/`。
- 如果程序以打包后的 `FlowDoc.exe` 运行：
  - `images/` 和 `attachments/` 会创建在 `FlowDoc.exe` 所在目录。
- 如果在开发环境运行：
  - `images/` 和 `attachments/` 会创建在项目目录。
- 若文件重名，会自动改名，避免覆盖已有资源。

### 标签与检索

- 每篇文档支持自定义多个标签。
- 标签会直接保存进 `.flowdoc` 文件元数据中，不是临时前端状态。
- 侧边栏文档库支持：
  - 按标签筛选
  - 按标题、路径、标签关键字搜索
  - 点击文档直接打开

## 文档格式

项目使用自定义的 `.flowdoc` 文档格式，本质上是一个 JSON 文件，核心字段如下：

```json
{
  "kind": "flowdoc",
  "version": 1,
  "updatedAt": "2026-04-18T09:00:00.000Z",
  "html": "<h1>文档标题</h1><p>正文内容</p>"
}
```

说明：

- `html` 字段保存规范化后的正文 HTML。
- 图片和附件不会以内嵌二进制写入文档，而是以相对路径节点引用程序目录下的资源文件。
- 这意味着 `.flowdoc` 文档适合轻量保存与版本控制，但分享给其他人时，需要同时带上对应的资源目录。

## 项目结构

```text
.
├─ index.html          # 应用主界面结构
├─ styles.css          # 编辑器与界面样式
├─ renderer.js         # 渲染进程逻辑，负责编辑器交互
├─ preload.js          # 预加载桥接，暴露 Electron 能力给前端
├─ main.js             # 主进程逻辑，负责文件、资源与窗口管理
├─ package.json        # 依赖、脚本与打包配置
├─ images/             # 运行时图片资源目录（已 gitignore）
├─ attachments/        # 运行时附件资源目录（已 gitignore）
├─ release/            # 打包产物目录（已 gitignore）
└─ tmp-electron-test/  # 临时测试目录（已 gitignore）
```

## 开发环境运行

### 安装依赖

```bash
npm install
```

### 启动开发版

```bash
npm start
```

## 打包

### 生成安装版和便携版

```bash
npm run dist
```

### 只生成便携版

```bash
npm run dist:portable
```

打包产物默认输出到 `release/` 目录，通常包括：

- `FlowDoc-1.0.0-x64-setup.exe`
- `FlowDoc-1.0.0-x64-portable.exe`
- `release/win-unpacked/FlowDoc.exe`

建议：

- 想固定到任务栏，优先使用安装版。
- 想免安装直接运行，可以使用便携版或 `win-unpacked` 目录下的可执行文件。

## 常用快捷键

- `Ctrl + S`：立即保存
- `Ctrl + Z`：撤销
- `Ctrl + Shift + Z`：重做
- `Ctrl + Y`：重做
- `Ctrl + A`：
  - 在代码块中第一次按下时全选当前代码块
  - 再按一次时全选整篇文档

## Git 说明

仓库默认忽略以下运行时或构建产物：

- `images/`
- `attachments/`
- `node_modules/`
- `release/`
- `tmp-electron-test/`

这样推送到 GitHub 时，只会上传源码和必要配置，不会把本地资源、依赖和安装包一起提交。

## 技术栈

- Electron

- 原生 HTML / CSS / JavaScript

## 当前实现侧重点

这个项目更偏向“本地桌面文档编辑器”的使用体验，重点放在：

- 简洁直接的所见即所得编辑
- 图片与附件的本地资源管理
- 类飞书文档的选区工具栏和块级插入体验
- 桌面端可打包、可安装、可固定到任务栏

如果后续要继续扩展，比较适合的方向包括：

- 表格节点
- 多文档标签页
- 全文搜索
- 导出 Markdown / PDF
- 更完整的快捷键体系
- 更强的附件预览能力
